Which tech stack do you want to build with?

This affects how fast we can ship and how easy it is to maintain later

A

Same as bengaluru.rent — Vanilla HTML + JS + Supabase

Single HTML file, no build step, deploy anywhere in minutes. Exact same architecture. Easier to copy patterns from bengaluru.rent directly.

Pros

  • Fastest to ship
  • Zero complexity
  • Cheap to host (Vercel/Netlify free)

Cons

  • Hard to scale features later
  • No component reuse
B

Next.js + Supabase

Modern React framework. Easier to add pages, SEO, auth, and email notifications later. More structured codebase.

Pros

  • Scales well as features grow
  • Built-in SEO (great for "Gurgaon rent" Google traffic)
  • Easier to add email, auth later

Cons

  • More setup time upfront
  • More complex than bengaluru.rent

💡 Recommendation: Option A to get live fast, then migrate to Next.js only if you need to. bengaluru.rent proves the vanilla approach works at scale.
