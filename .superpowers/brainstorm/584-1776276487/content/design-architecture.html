Design — Section 1: Architecture

How the project is structured. Each file has one job.

📁 Project Structure
ncr-rent-map/
index.html — shell: loads map, imports all JS modules
style.css — all UI styles (dark blue theme)
js/
config.js — Supabase URL/key, Google Maps key, constants
map.js — Google Maps init, clustering, pin rendering
pins.js — load pins, submit pin, edit, delete
filters.js — filter panel logic (BHK, price, gated, metro km)
stats.js — live stats modal, area stats, leaderboard
metro.js — Delhi Metro all 9 lines GeoJSON + stations
aqi.js — AQI overlay: fetches WAQI API, draws colour zones
expressways.js — NH-48, Dwarka Exp, Yamuna Exp polylines
sectors.js — NCR sector/area data (Gurgaon, Noida, Delhi zones)
auth.js — device ID, IP fetch, ownership tracking
realtime.js — Supabase Realtime subscription for live pins
supabase/
schema.sql — all tables, RLS policies, RPCs
functions/get-pins/ — Edge Function: serves pins with rounded coords
